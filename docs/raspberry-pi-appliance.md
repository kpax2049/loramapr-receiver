# Raspberry Pi Appliance Path (Deprecated)

The custom LoRaMapr Receiver Raspberry Pi appliance image path is currently
**deprecated/paused** and is not part of the active supported release flow.

## Use This Path Instead

Use official Raspberry Pi OS Lite + package install:

- [Linux/Pi Existing-OS Install Path](./linux-pi-distribution.md)

## What Deprecation Means

For now, LoRaMapr Receiver does not treat the custom Pi image as a first-class
public path:

- no active release artifact generation for Pi image
- no active publish/verify requirement for Pi image
- no first-class reviewer test path
- no public recommendation as the default Pi install method

Existing historical appliance documentation and scaffolding remain in-repo for
internal reference only.
